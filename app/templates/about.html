About Vision : The LLM Council

What is Vision : The LLM Council?

Vision : The LLM Council is a multi-LLM collaborative reasoning system that simulates an expert committee solving problems together. Instead of relying on a single AI model, it orchestrates multiple models with different roles to generate higher-quality, more reliable answers through structured deliberation.

The Deliberation Process

Stage 1: Role-Based Responses

Each council member (LLM) receives the user's query along with their assigned role description. They generate responses from their unique perspective:

  • Researcher: Provides factual background and explanations
  • Critic: Identifies weaknesses and challenges assumptions
  • Creative Thinker: Suggests innovative ideas and alternatives
  • Practical Advisor: Focuses on actionable implementation

Stage 2: Anonymous Peer Review

Each model reviews responses from other council members without knowing which model generated them. This ensures unbiased evaluation based on content quality rather than model reputation.

Reviewers score each response on:

  • Accuracy (30%)
  • Completeness (25%)
  • Reasoning Quality (25%)
  • Clarity (20%)

Stage 3: Multi-Round Improvement

After receiving peer feedback and seeing other perspectives, each council member has the opportunity to revise and improve their response. This simulates how human experts refine their opinions after hearing diverse viewpoints.

Stage 4: Weighted Consensus Scoring

Peer review scores are aggregated into weighted scores for each response. Higher-scoring responses have more influence on the final synthesis. The system also calculates an overall consensus score indicating how much agreement exists among council members.

Stage 5: Verifier Validation

A dedicated Verifier LLM analyzes all responses to detect:

  • Hallucinations (fabricated information)
  • Factual errors
  • Logical inconsistencies
  • Contradictions between responses

The verifier produces a reliability report that informs the final synthesis.

Stage 6: Chairman Synthesis

The Chairman LLM receives all role-based responses, peer review scores, and the verification report. It synthesizes these into a comprehensive final answer that:

  • Weighs perspectives by their merit
  • Addresses verified concerns
  • Highlights consensus and disagreements
  • Provides a clear, actionable conclusion

Key Features

🎭 Role-Based Diversity

Different roles ensure diverse perspectives on every query.

🔒 Anonymous Review

Unbiased peer review without model identity influence.

🔄 Iterative Improvement

Multiple rounds allow refinement based on feedback.

⚖️ Weighted Scoring

Merit-based contribution to final synthesis.

✅ Error Detection

Dedicated verification catches hallucinations and errors.

📊 Transparency

Full reasoning trace shows how answers were formed.

API Access

Vision : The LLM Council provides a full REST API for programmatic access. Visit /api/docs for interactive API documentation.

Key endpoints:

  • POST /api/deliberate - Run full deliberation
  • GET /api/session/{id} - Get session data
  • GET /api/config - Get configuration
  • GET /api/roles - Get available roles
